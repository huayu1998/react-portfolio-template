# Heading One

**Distributed systems** look elegant in diagrams but behave unpredictably in production. After working on high-throughput services, one truth stands out: failures are not exceptions—they are guarantees.

## This is Heading Two

At scale, problems shift from correctness to availability. Concepts like eventual consistency and replication lag become tangible. Choosing between consistency and availability is no longer theoretical; it directly impacts user experience.

### This is Heading Three

Observability is the backbone of reliability. Metrics, logs, and traces are not optional—they define your ability to debug. Systems without visibility are effectively black boxes.

const fetchWithRetry = async (fn, retries = 3) => {
  try {
    return await fn();
  } catch (err) {
    if (retries <= 0) throw err;
    return fetchWithRetry(fn, retries - 1);
  }
};
